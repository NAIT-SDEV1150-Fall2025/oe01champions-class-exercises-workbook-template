This is the SIXTH page that explain the concept of VARIABLES in JavaScript.

Variables can be called as containers that can store the data values in it. Variables must be identified with unique names.

JavaScript is case sensitive. SO, please be very carefull when defines the variable/function names and uses those at various places of program.

JavaScript Identifiers

Identifiers are used to name variables and keywords, and functions.

The rules for defining legal names are the same as most programming languages and those are:

A name can have letter (A-Z or a-z), A number (0-9), A dollar sign ($) or An underscore (_).

****Numbers are not allowed as the first character in names. Because, this way JavaScript can easily distinguish identifiers from numbers.

JavaScript keywords - that tells what action to be performed. In other terms, declaration of variables

  • const
  • Var (used in old JavaScript and not recommended)
  • automatically (used in old JavaScript and not recommended)

After you declared your variable, you must also let your machine know that what type of data is going to be stored in these variables/containers.

JavaScript DATA TYPES - A JavaScript variable can hold 8 types of data

  • String - A text of characters enclosed in quotes (double or single quotes)
  • NUMBER - A number representing a mathematical value, It has min and max values, when required value does not fit the max value then use BIGINT
  • BIGINT - A number representing a large integer
  • BOOLEAN - A data type representing true or false
  • OBJECT
  • UNDEFINED - A variable with no assigned value
  • NULL
  • SYMBOL
STRINGS

A string is used to store a text value. It is a sequence of characters. There are different ways to declare a string: Double quotes Single quotes Backticks: special template strings in which you can use variables directly

The main difference between single quotes and double quotes is that you can use single quotes as literal characters in double-quoted strings, and vice versa. If you declare a string with single quotes, the string will end as soon as a second quote is detected, even if it's in the middle of a word. However, if you declare the string using double quotes, it will not end the string as soon as it hits the single quote, because it is looking for another double quote.

In a string using backticks, you can point to variables and the variable's value will be substituted into the line.

ESCAPE CHARACTERs

Say we want to have double quotes, single quotes, and backticks in our string. We would have a problem, as this cannot be done with just the ingredients we have now. There is a special character that can be used to tell JavaScript, "do not take the next character as you normally would." This is the escape character, a backslash.
